제가 웹개발을 시작한지 벌써 7년이 다 되어 가네요. 물론, 그 시간동안 하루종일 개발만 공부한 건 아니지요. 놀거 다 놀면서 취미로 시작했다가 이제는 개인 웹사이트를 만들고 서비스도 운영하고 있는 중입니다.

그 과정에서 제가 축적한 노하우를 여러분께 나누어 드리고자 이렇게 간단한 블로그를 하나 만들게 되었어요. 오늘부터 4강에 걸쳐 저희가 배우게 될 내용은:

이 블로그와 같은 웹사이트를 바닥부터 만들어서 배포하는 방법입니다.

내용이 생각보다 많아서 놀라실 수 있는데, 제가 4강, 즉 빠른 사람에게는 4시간, 그리고 완전 초보에게는 10시간 정도의 시간 안에 웹사이트를 만드는 방법을 알려드릴게요.

그 전에 1강에 대한 소개를 하겠습니다.

head태그는 이 페이지의 주요 내용 부분은 아니고 브라우저 탭에 보이는 이름, 아이콘 등과 같은 부수적인 정보를 정의내리는 곳이에요.

그리고 body부분이 저희가 주로 다루게 될 부분이고요.

html boilerplate는 너무 어려운 태그들이 많으니 필요한 것만 빼고 다 지워버리겠습니다. 그리고 title태그에 이 페이지의 이름을 넣고, description 내용을 한번 채워봤어요.

마찬가지로 body태그 안에 앞서 작성한 hello world를 다시 넣어줬어요. 그 결과는 어떨까요?


hello world

네, 이런 식으로 엄청 간단한 페이지가 나오겠죠? 하지만...실제 브라우저로 이 페이지를 확인하면 변한게 있어요.

눈치 빠르신 분들은 바로 아셨겠죠. 브라우저의 탭 부분에 금방 변경한 title의 내용이 나와요. body는 변화가 없고요.

결국, head부분에서는 메인 창 이외의 사이트 정보를 수정하는 곳이라고 생각하면 될 것 같아요.

더 나아가서 body태그 안에서 사용할 수 있는 다양한 기본 태그들을 조금 더 설명드릴게요.

1. div, span, p 태그

div태그는 아마 가장 많이 사용되는 html태그 중 하나에요. div는 division의 줄임말 같은데 html에서 블락(block) 레벨의 요소가 되요. 브라우저가 html태그들을 화면상에서 보여줄 때는 하나의 태그씩 차곡차곡 쌓아가면서 보여주는데 이는 그 태그들이 블락 요소들이기 때문이에요.

한번 예를 들어볼게요.

이렇게 div태그 3개를 body태그 안에 만들어 보세요. 그리고 브라우저를 보면 아무 변화가 없을 거에요. div태그는 아무 높이를 설정해주지 않으면 화면상에서 보이지 않기 때문이지요.

그러면 어떻게 해야 div태그들을 활성화시킬까요? 바로 텍스트를 넣어주면 되요. div는 따로 명시된 넓이나 높이가 없다면 자신의 내용물과 같은 넓이나 높이를 자동으로 가져가게 되요. 내용물을 채워볼게요.
div 1
div 2
div 3

여러분도 이렇게 코드를 치고 브라우저를 봤더니 저렇게 텍스트가 하나씩 차곡차곡 쌓였나요? 저런 행동 방식 때문에 div는 block element라고 불러요. 그러면 저렇게 블락처럼 안 쌓이게 하는 방식도 있을까요? 네, inline element를 사용하면 되요!

그 대표적인 예가 바로 span태그에요. span태그는 다른 태그의 아래로 위치하게 되는게 아니라 옆에 위치하게 된다고 inline 요소라고 불리죠.

이렇게 코드를 짜면 그 결과는:

div 1span 1
div 2span 2
div 3span 3

이렇게 보여요. div태그끼리는 서로 열의 방향으로 움직였다면, span은 행의 방향으로 움직이죠?

여기서 태그 하나만 더 배우고 다음 중요한 태그로 넘어갈게요. 이번에 볼 태그는 p태그에요. p는 paragraph의 줄임말로 추정되고, div와 마찬가지로 block 레벨이라고 보시면 되요.

p태그는 사용하기 쉽기 때문에 제가 실제로 보여드리지는 않을게요. 직접 코드에 p태그를 추가하고 브라우저에서 그 결과를 확인해보시면, div태그와 차이를 이해 못할 수도 있어요.

여기서 중요한 개념 하나 짚고 넘어갈게요! Semantic HTML

시멘틱 하다는 것은 '의미'를 기반으로 한다는 것이에요. 그렇다면 Semantic HTML이라는 것은 html을 의미를 나타내기 위해서 사용한다는 것이 되는거죠.

div와 p태그의 작동 원리는 같지만 결국은 이름만 다른 거잖아요. 이는 html의 구조를 봤을 때 div는 레이아웃 상 division을 나타내는 태그로 활용되고, p는 컨텐츠를 담고 있는 태그로 활용되기 때문에 분명 구분지어 사용되어야 하죠.

2. h1, h2, h3, h4, h5, h6 태그

이번에는 h1 ~ h6태그들에 대해서 알아볼게요. h는 heading의 줄임말이에요. 보통 해딩은 제목이나 부제목을 나타낼때 사용하잖아요. h태그도 마찬가지에요.

웹사이트 상에서 제목이나 부제목을 나타낼때 h태그를 사용하면 되고, h태그는 이제 div, p와는 다르게 스타일링이 추가되어서 보여요! 한번 확인해볼까요?

이렇게 코드를 작성하고 브라우저를 확인해주세요.

p태그와는 확실히 다르죠?

브라우저를 확인해보시면 h1부터해서 h6까지 글자 크기가 점점 작아지는걸 볼 수 있죠. 자신의 취향에 맞게 적당한 사이즈와 두께의 해딩을 선택해서 사용하면 되요.

3. ul, ol, li, em, strong, br, hr 태그

이번에는 또 기본적이지만 정말 많이 사용되는 태그 여러개를 배워볼게요. 아, 그전에 스타일 넣는 방법이 아마 궁금할거 같은데요. 스타일링은 CSS를 배우면서 할거에요~ 그러니깐 걱정마시고 우선은 HTML을 배우고 바로 넘어가죠.

웹사이트를 개발하다보면 리스트 형식의 글을 쓸 때가 있을 거에요. 그런 경우 ul, ol, li 태그들을 사용하고,

글자를 italic 시키거나 bold처리하고 싶을 때는 em, strong태그를 사용하면 되요.

그리고 웹사이트 상에서 빈 공간/여백을 나타내고 싶을 때는 br태그를, 그리고 수평선을 긋고 싶을 때는 hr태그를 사용하면 되요.

CSS 적용하는 다른 방법

CSS는 위에서 본 것과 같이 다른 파일에서 관리하는 방식이 있고, HTML파일 내에 바로 정의하는 방식이 있어요.

물론 따로 관리하는게 훨씬 코드 작성하기 편하고 추후에 유지보수도 쉬워지겠지만, 지금 저희는 연습하고 있잖아요? 그러니깐 하나의 파일로 모든 코드를 테스트해보는 것도 좋은 생각이에요.

다시 index.html 파일을 수정할게요. 이번에는 금방 정의한 link태그를 주석처리 했어요. 위에 보이는 것과 같이 코드 줄을 감싸주면 주석처리가 되어 브라우저에서는 무시를 하고 넘어가요.

일일이 저렇게 치기 힘들기 떄문에 모든 텍스트 에디터의 공통된 기능으로 주석처리 할 코드 줄을 선택한 후에 CMD + / (윈도우: CTRL + /)을 누르면 자동으로 주석처리 될거에요.

어떤가요? live server로 한번 확인해보셨나요? css파일을 연결시킨 것과 같은 효과죠? 저런 식으로 css 코드가 길지 않다면 저렇게 하는 것도 나쁘지 않지만, 곧 아시게 되겠지만 코드가 상당히 길어질 거에요...그래서 처음부터 그냥 css 파일을 따로 관리하는 방향으로 블로그글을 진행할게요.

사실 css를 적용시키는 다른 방식이 하나 더 있어요. 그것도 보여드리고 넘어갈게요.

간단하게 생각해서 style태그 안의 css가 body 색상을 lightblue라고 정의내렸지만, 그 아래에 있는 body태그 안에 있는 style속성이 red라고 정의를 내림으로써 body태그 안에 있는 style이 우선순위로 적용된거죠.

텍스트의 크기, 폰트, 색상을 변경시키는 방법은 이미 봤어요. 바로 font-size, font-family, color를 사용하는 방법이었죠.

텍스트 크기에 대해 배우기 전에 웹브라우저의 유닛 크기에 대해 알아볼게요. 1px이 컴퓨터 화면의 최고 기본 단위인건 아실거에요. 그럼 1px이 어느정도 크기인지 확인해볼까요? 저 폰트를 1px 크기로 변경해보죠.

p {
    font-size: 1px;
}

그나저나 1px 생각보다 그렇게 작지는 않네요. 저는 거의 점일거라 예상했는데...

css에서 사용하는 유닛 중에 em, rem, vh, vw가 있어요. 제가 소개드릴 단위는 rem이에요. 실제로 저는 현재 px, rem만 사용하고 있거든요.

    
html {
    font-size: 62.5%; /* 16px이 100%, 62.5%는 10px */
}

p {
    font-size: 1px;
}

이렇게 위와 같이 html태그에 font-size를 지정해주면 1rem은 저 폰트 사이즈와 동일하게 되요. 그 이유는 rem은 최상위 태그의 폰트 사이즈를 기본값으로 가져 가기 때문이고, html태그가 모든 태그의 최상위 태그이기 때문이죠.

기본적으로 html태그의 폰트 크기는 16픽셀이에요. 거기서 62.5%를 취해주면 10px이 되요. 그래서 저는 1rem을 10px로 인식하고 쉽게 사용하려고 저렇게 미리 정의내리고 시작해요.

앞으로 저는 rem 단위만 사용할게요!

이제는 텍스트 크기 외 추가적으로 폰트와 색상도 변경해볼게요.

    
p {
    font-size: 1px;
    font-family: 'Nanum Gothic', sans-serif;
    color: #6c5ce7;
}

이번에는 간단한 텍스트 정렬을 해볼게요. 텍스트 정렬은 text-align을 사용해서 할 수 있고, 사용할 수 있는 속성은: left, center, right을 사용하시면 되요.

우리가 예쁘게 만들 텍스트

이렇게 텍스트가 가운대 정렬되는게 보이죠? 마지막으로 이건 자주 사용되는 기능은 아니지만, 그래서 고급 기능이라고 생각해도 되겠네요. text-shadow에 대해서도 간단하게 설명드릴게요.

    
p {
    font-size: 1rem;
    font-family: 'Nanum Gothic', sans-serif;
    font-weight: bold;
    color: #6c5ce7;
    text-align: center;
    text-shadow: 3px 3px 0px #f2a063;
}

이번에는 font-weight도 추가했고, text-shadow도 함께 추가했어요. font-weight는 normal(400), bold(700), bolder, lighter처럼 영어로 정의내려줘도 되고, 100부터 900까지 100단위로 900이 가장 두꺼운 글씨로 정의내릴 수도 있어요.

그리고 text-shadow는 좀 복잡하죠? 3px 3px 0px #f2a063. 왜 이렇게 4개의 옵션이 있을까요?

text-shadow는 [x거리] [y거리] [흐림정도] [색상]과 같이 4개의 옵션을 추가해서 정의내릴 수 있어요. x거리를 키울 수록 텍스트 그림자는 오른쪽으로 더 나오게 되고, y거리를 키울 수록 아래로 내려오게 되죠. 반대로 -를 붙여서 정의내리면 반대로 이동해요.

우리가 예쁘게 만들 텍스트

결과물이 보이나요? 물론 마음에 들지는 않지만 저것도 마음에 들게 할 수 있어요. 바로...

    
p {
    font-size: 2rem;
    font-family: 'Nanum Gothic', sans-serif;
    font-weight: bold;
    color: #6c5ce7;
    text-align: center;
    text-shadow: 3px 3px #f2a063,
                 2px 2px #f2a063,
                 1px 1px #f2a063;
}

위와 같이 text-shadow를 여러번 정의해주는 방식이에요. 이렇게 하면 그림자가 두꺼워져서 동떨어진 곳에 갑자기 그림자가 생기는게 아니라 텍스트와 한 몸인 것처럼 보이죠.

여러분은 이제 css로 텍스트에 효과를 주는 거의 모든 방법에 대해 익혔어요. 이제는 박스 모델에 대해서 배워볼게요.

The Box Model

저~~ 위에서 div태그는 블락 요소라고 했던 것 기억하나요? 블락이기 때문에 한 줄을 전부 차지하고 다음 블락 요소는 그 아래에 위치할 수밖에 없죠.

index.html에서 이렇게 정의내린 것 기억하나요?

그 결과는 브라우저에서 아무 것도 안 보이는 거였죠. 하지만 여기서 이제 css를 알기 때문에 div태그의 넓이와 높이를 정의내려줄 수 있어요.

    
div {
    width: 100%;
    height: 3rem;
    background: #6c5ce7;
}

이렇게 index.css에서 div태그의 요소들에 모두 width는 100퍼센트로 (여기서 px, rem을 사용해서 자신이 원하는 넓이로 정의해줘도 되요! 저는 그냥 브라우저를 꽉 채우게 하고 싶어서 100%로 했어요~) 그리고 높이는 3rem으로 정의내렸어요.

여기서 추가로, div태그의 background는 div태그가 차지하고 있는 공간만큼만 배경에 색상을 입혀주죠. 아래에서 한번 확인해볼게요.

음...뭔가 보라색 공간이 생긴거 같긴한데, 저는 분명히 div태그 3개를 만들었잖아요. 왜 하나죠?

여기서 그 유명한 박스 모델이 나와요. HTML의 모든 태그들은 일정한 넓이와 높이를 가지고 브라우저 안에서 공간을 차지하고 있어요. 근데 그 공간은 모두 박스의 형태죠. 그리고 다음 태그 요소와 사이에 아무 공간도 없어요! 그렇기 때문에 아무리 다음 태그로 넘어가도 만약 같은 색상의 배경색이라면 그 경계가 뚜렷하지 않죠.

그 경계를 눈으로 확인하기 위해서 div의 배경색을 모두 다르게 해볼게요. class(클래스)라는 css 선택자를 사용하면 각 div에 클래스를 지정하여 따로 css를 적용할 수 있어요.

index.html에서 저는 일부러 첫번째, 마지막 div에게 pink-div라는 클래스명을 주고, 두번째 div는 purple-div라는 id값을 줬어요.

사실 class나 id 모두 비슷하지만, 보통은 여러 요소에 공통된 디자인을 적용시키고 싶을때 class를, 하나의 타겟된 요소에 디자인을 적용시키고 싶은 때 id를 사용해요.

    
div {
    width: 100%;
    height: 3rem;
}

.pink-div {
    background: #fd79a8;
}

#purple-div {
    background: #6c5ce7;
}

index.css에서 제가 뭘 했는지 잘 봤나요? 클래스에 css를 적용시키기 위해서 저는 pink-div라는 클래스의 css 공간을 정의내렸지만, 그냥 pink-div라고 하지 않고, .pink-div라고 했어요.

클래스를 지정할 때는 .을 붙이고, id를 지정할 때는 #을 붙여줘야 해요. 그럼 그 결과를 확인해볼게요?

이제 완벽하죠? 이렇게 div 태그의 공통된 디자인은 태그 전체에 css를 정의내리고, 색상만 변경하기 위한 클래스와 아이디명을 div태그에 줘서 원하는 디자인을 적용시켰어요.

padding: [padding top/위] [padding right/오른쪽] [padding bottom/아래] [padding left/왼쪽]

padding: 1rem;

padding: 1rem 1rem 1rem 1rem;

padding: 1rem 1rem;

은 모두 같은거죠.

여기서부터는 살짝 고급 단계의 맛을 보기 시작할게요. Flexbox를 배우기 시작할건데 CSS를 하는 모든 사람들이 처음에는 아무도 이해 못하고 시작했을 거에요. 저도 정말 이해하는데 오래 걸렸거든요.

하지만! flexbox를 이해하면 너무나 유용하기 때문에 힘들어도 꼭 앉아서 이 부분만큼은 끝까지 따라해보길 바라요.

특히 flexbox가 어렵다고 과거에 웹사이트 레이아웃을 잡는 방법을 배워보겠다고 나서면 그때서야 알게 될거에요. '아...flexbox가 쉬운거구나.'

여기서 저는 flexbox를 이해하기 위해 필요한 기초툴들을 하나씩 소개할게요. 처음에는 느리고 이해 안 되겠지만, 결국 모두 하나가 되면서 완벽하게 이해될거에요.

Flexbox를 사용하면 웹개발을 할때 복잡한 레이아웃과 반응형 디자인(모바일 유저를 고려한 웹사이트 디자인이에요!)을 적용하기 쉬워져요.

Intro

웹개발할때 레이아웃이 왜 중요한가요?

여러분, 위에서 배운 내용으로 사이드바나 가로로 쭉 뻗어있는 버튼들을 디자인할 수 있나요?

아직은 못하죠? 왜냐면 박스 모델에서 배운 내용으로는 HTML의 모든 요소들은 새로로 쌓이게 되어있잖아요.

그래서 네이버같은 복잡한 사이트를 개발하려면 HTML 요소들을 사이드로 보내는 방법, 아니면 훨씬 복잡한 위치 배정 기술이 있어야 해요. 여기서 flexbox가 도움이 되는거죠.

이해가 안 되죠? 그러면 두말없이 코드부터 보여드릴게요.

새로운 html파일을 하나 만들어볼게요. 저는 파일명을 flexbox.html이라고 할게요. head부분은 본인이 원하는대로 작성해도 되고, 아예 없어도 아래의 예시를 구현하는데 문제 없을거에요!

여기서부터 배우게 될 CSS 속성들은 무조건 display: flex가 설정되어 있어야지만 작동해요! 꼭 주의하세요!

flex-direction

flex-direction을 사용하면 main axis의 방향을 설정할 수 있어요. main axis란, flex 컨테이너의 방향을 얘기해요. 위에서 display: flex를 설정해주면 flex 아이템들이 가로로 배치하게 되는걸 봤잖아요.

이 축의 방향을 main axis라고 부르고, 원하는대로 변경할 수 있죠.

flex-direction: row;

flex-direction: column;

flex-direction: row-reverse;

flex-direction: column-reverse;

이렇게 main axis의 방향은 행, 열, 역행, 역열의 방향으로 설정해줄 수 있어요.

여기서 또 중요한 축이 하나 나오는데, 바로 cross axis에요. Cross axis는 어렵게 생각하지 마시고, main axis의 수직 관계라고 생각하면 되요. 만약 main axis가 가로였다면, cross axis는 새로축이 되는거죠.

하나씩 확인해볼까요? 꼭 직접 쳐서 확인해보길 바랄게요.

우선 결과부터 보여드릴게요. 첫번째 줄부터 시작해서 row, column, row-reverse, column-reverse를 적용시켜보고 각 구역별 구분을 더 잘 짓기 위해서 border-bottom을 추가했어요. 코드를 확인해볼게요.
flexbox.html
    
...

.flex-div {
    display: flex;
    border-bottom: 1px solid #6c5ce7;
}

.row {
    flex-direction: row;
}

.column {
    flex-direction: column;
}

.row-reverse {
    flex-direction: row-reverse;
}

.column-reverse {
    flex-direction: column-reverse;
}

.flex-item {
    width: 2rem;
    height: 2rem;
    margin: 0.5rem;
}

.flex-item.blue {
    background: #0984e3;
}

.flex-item.purple {
    background: #6c5ce7;
}

.flex-item.green {
    background: #00cec9;
}
flexbox.css

복잡해 보이지만 사실은 되게 간단하죠? flex-div를 여러개 만들고 각각 flex-direction을 다르게 설정해줬어요. 그리고 flex 컨테이너 안의 아이템들이 어떻게 배치되는지 더 잘 보기 위해서 blue, purple, green 클래스를 만들어서 백그라운드 색상을 변경해줬어요.

이 코드를 보니 flex-direction이 적용되는 원리를 이해하겠나요?
